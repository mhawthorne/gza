# Documentation Review

## Overall Summary

Documentation covers core workflows well but has fallen significantly out of sync with the CLI. 8 commands are completely undocumented, the PyPI README uses nonexistent commands, and several documented behaviors are wrong (task type `task` removed, `status` command misdescribed, wrong Gemini CLI package name).

## Accuracy Issues Found

| Issue | Location | Details |
|-------|----------|---------|
| PyPI README uses nonexistent commands | README_PYPI.md:28-35 | `gza task add`, `gza worker start`, `gza task list` don't exist. Should be `gza add`, `gza work`, `gza ps`/`gza history` |
| Skills doc uses old command name | docs/skills.md:8-14 | References `gza claude-install-skills`. Actual command is `gza skills-install` |
| Wrong Gemini CLI package | docs/quickstart.md:28 | `npm install -g @anthropic-ai/gemini-cli` ŌĆö should be `@google/gemini-cli` (Google, not Anthropic) |
| `task` type still documented but removed | docs/configuration.md:373,799 | `add --type` lists `task` but CLI only accepts `{explore,plan,implement,review,improve}`. DB migration v19ŌåÆv20 renamed `task` to `implement` |
| `task` type in bulk import example | docs/examples/bulk-import.md:27,45,89 | Uses `type: task` which is no longer valid |
| `status` command misdescribed | docs/configuration.md:515-519 | Documented as "Show tasks in a group" but it's actually "List running workers (alias for ps)". The `group` command shows tasks in a group |
| `max_steps` missing from config table | docs/configuration.md | Appears in resolution precedence but not in the Optional Configuration table. `DEFAULT_MAX_STEPS = 50` exists in code |
| Default task type not stated | docs/configuration.md, quickstart.md | Default type for `gza add` is `implement` (not `task`), but docs don't clearly state this |
