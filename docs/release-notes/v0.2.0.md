# Release Notes: v0.1.0 â†’ v0.2.0

This release brings significant improvements to the `gza` workflow tool, including new merge options, enhanced CLI usability, background task support, and better Claude Code integration through skills and logging improvements.

## Features

- **Multiple task support in `gza work`** - Now supports working on multiple task IDs in a single command
- **Background flag for resume/retry** - Added `-b/--background` flag support to `resume` and `retry` commands for running tasks in the background
- **Release notes generation** - New script/command to automatically generate release notes from git history
- **Claude Code skills** - Added new slash command skills for adding gza tasks and retrieving task information
- **Review docs skill** - New skill for reviewing documentation

## Improvements

- **Merge command enhancements**
  - Added `--squash` option for squash merging branches
  - Added `--rebase` option for rebasing before merge
  - Improved merge to abort on failure, restoring clean working state
  - Fixed merge to only check for tracked file changes
- **Enhanced Claude provider logging** - Now shows detailed information for better debugging
- **Plan/explore/review task output** - Claude now writes results to files for plan, explore, and review tasks
- **CLI refactoring** - Converted `project_dir` from a positional argument to an option for better usability
- **Helpful command hints** - Added command suggestions at the end of `gza work` output
- **Next steps messaging** - Updated completion message to include `gza merge` command

## Bug Fixes

- Fixed `has_changes()` in `git.py` to properly detect untracked files
- Fixed rebase script issues
- Resolved CRLF line ending issues and added `.gitattributes` to prevent them

## Documentation

- Improved documentation throughout
- Renamed `gza` to `gza-agent` in uv installation docs
